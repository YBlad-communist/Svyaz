# Security & Reliability Changelog

## Критические исправления безопасности

### 1. CSRF-защита (было: отсутствует, стало: полная)
**Проблема:** CSRF-токен генерировался и вставлялся в формы, но **ни один маршрут не проверял его**. Любой сайт мог отправить POST-запрос от имени авторизованного пользователя.

**Исправление:**
- Добавлен декоратор `@csrf_required` и функция `validate_csrf_token()`
- Токен проверяется из `request.form['_csrf_token']`, `X-CSRFToken` header, или JSON body
- Используется `secrets.compare_digest()` для защиты от timing-атак
- Все POST/PUT/DELETE маршруты защищены декоратором
- При нарушении CSRF: JSON-запросы получают `403`, обычные — редирект с flash-сообщением

### 2. HTML-санитизация (было: `html.escape`, стало: `nh3.clean`)
**Проблема:** `html.escape()` экранировал ВСЕ HTML-теги, включая легитимные `<code>`, `<pre>`, `<strong>`, `<a>` — разработчики не могли форматировать код в постах.

**Исправление:**
- Заменён на `nh3.clean()` (современный форк bleach)
- Разрешены безопасные теги: `<code>`, `<pre>`, `<strong>`, `<em>`, `<a>`, `<ul>`, `<ol>`, `<li>`, `<blockquote>`, `<h1>`-`<h6>`, `<table>`, `<img>` и др.
- Запрещены: `<script>`, `<iframe>`, `<object>`, `<embed>`, `<form>` и все обработчики событий (`onclick`, `onload` и т.д.)
- Запрещены протоколы `javascript:`, `data:` в ссылках

### 3. Удалены фейковые `encrypt_text`/`decrypt_text`
**Проблема:** Функции «шифрования» были заглушками (`return text`). Контент хранился в БД без шифрования, но код создавал ложное ощущение безопасности.

**Исправление:**
- Функции удалены
- Контент хранится в sanitized-виде напрямую
- Поиск по постам теперь работает корректно (раньше искал по «зашифрованному» тексту)

### 4. `before_request` больше не делает `db.session.commit()`
**Проблема:** Каждый запрос от авторизованного пользователя вызывал `commit()`, что:
- Прерывало транзакции в маршрутах
- Могло привести к потере данных при ошибках
- Создавало unnecessary writes на каждый запрос

**Исправление:**
- Заменено на `db.session.flush()` (обновляет last_activity без коммита)
- Добавлен `@app.teardown_appcontext` с `db.session.remove()` для гарантированной очистки

### 5. Rate limiting расширен
**Добавлены лимиты:**
| Маршрут | Лимит |
|---------|-------|
| `/register` | 10/мин |
| `/login` | 10/мин |
| `/post/create` | 30/час |
| `/post/<id>/like` | 60/час |
| `/post/<id>/comment` | 60/час |
| `/idea/create` | 20/час |
| `/idea/<id>/like` | 60/час |
| `/idea/<id>/join` | 10/час |
| `/user/<username>/follow` | 30/час |
| `/upload_avatar` | 10/час |
| `/channel/create` | 10/час |
| `/channel/<name>/post` | 30/час |
| `/api/chat/<id>/send` | 60/час |
| `/api/chat/create` | 20/час |
| `/create_group` | 10/час |
| `/channel/<name>/invite/create` | 20/час |

### 6. Проверка прав доступа
**Исправлено:**
- `idea_delete`: теперь редиректит с flash вместо JSON (консистентность)
- `channel_page`: приватные каналы недоступны не-участникам
- `idea_approve_join`: нельзя одобрить свою собственную заявку
- `channel_post_like`: убран ручной `likes_count` (race condition), используется COUNT из БД
- Все `get_or_404` заменены на `db.session.get()` + проверка (SQLAlchemy 2.0 compatible)

### 7. Безопасность файлов
**Исправлено:**
- `safe_save_file` теперь поддерживает параметр `allowed_types` ('image', 'video', 'file')
- Загрузка аватаров принимает только изображения
- Path traversal защищён через `secure_filename` + проверка абсолютного пути
- MIME-тип проверяется по заголовкам файла, не по расширению

### 8. Валидация данных
**Добавлено:**
- `validate_email()` — regex-проверка формата email
- `validate_username()` — проверка длины и допустимых символов
- Минимальная длина пароля увеличена с 6 до 8 символов
- `generate_password_hash` теперь использует `pbkdf2:sha256` с `salt_length=16`
- `expires_hours` в invite-ссылках: минимум 1 час, максимум 720

### 9. Обработка ошибок
**Добавлено:**
- Глобальный `@app.errorhandler(Exception)` — не раскрывает детали клиенту
- `db.session.rollback()` в 500-обработчике
- Логирование ошибок через `logger.error(..., exc_info=True)` — только на сервер
- Защита от leak чувствительных данных в traceback

### 10. Приватные каналы
**Добавлено:**
- `channel_page` проверяет membership для приватных каналов
- Не-участники получают редирект на список каналов

## Как запустить тесты

```bash
# Установка зависимостей
pip install -r requirements.txt

# Запуск всех тестов
pytest test_app.py -v

# Запуск с покрытием
pytest test_app.py -v --cov=app --cov=module

# Запуск конкретного класса тестов
pytest test_app.py -v -k TestCSRF
pytest test_app.py -v -k TestRegister

# Запуск с выводом print-отладки
pytest test_app.py -v -s
```

Тесты используют SQLite in-memory (`sqlite:///:memory:`) и не требуют PostgreSQL или Redis.

## Миграция БД

### Новая колонка `ideas.project_type`
После обновления кода запустите миграцию для добавления колонки:

```bash
python migrations/versions/add_project_type.py
```

Это добавит `project_type VARCHAR(30) DEFAULT 'other'` в таблицу `ideas`.
Все существующие идеи получат тип `other`.

Остальные изменения обратно совместимы:
- Новые функции (`validate_email`, `validate_username`) — только в Python-коде
- `nh3.clean` заменяет `html.escape` — результат хранится в тех же колонках
- Удаление `encrypt_text`/`decrypt_text` — контент уже хранится в незашифрованном виде (функции были заглушками)
- Новые декораторы (`@csrf_required`) — не меняют схему БД
